Nota estratégica

Por qué esto vale más que los tours

El turista cultural paga más por experiencias auténticas. Un taller de Coco ART con Breda Sky no lo encuentra en Booking ni en ninguna OTA. Solo existe en GuanaGO porque solo GuanaGO tiene acceso a ese ecosistema Raizal. Esto justifica el sello "experiencia exclusiva" y precios 2–3x más altos que un tour estándar.

Bono GuanaPoints

Completar una experiencia cultural = +100 GP. Es el bono más alto del ecosistema. Incentiva explícitamente al turista a elegir cultura sobre turismo masivo.

Base de datos unificada por tipo de aliado

Airtable como maestro de datos → Make sincroniza a Firestore → App lee desde Firestore. Un registro en Directorio_Mapa = un aliado. Tablas extensoras por tipo.

Regla de diseño

Directorio_Mapa es el registro maestro. Todos los demás (Alojamiento, ServiciosTuristicos_SAI, Menu_Restaurante) son tablas linked por Aliado_ID.

Blocker #1 actual

31 registros en Directorio_Mapa, todos con ID_Slug vacío. Sin slug, ningún microsite tiene URL. Esta es la tarea de mayor impacto antes del lanzamiento.

Blocker #2 actual

~18 registros con teléfono 5555555. Limpiar antes de hacer público el directorio para no mostrar datos falsos al turista.

Script para poblar ID_Slug

Patrón: tomar Nombre_Comercial, convertir a lowercase, reemplazar espacios por -, eliminar tildes y caracteres especiales. Ejecutar en Make.com o directamente vía Airtable API.

// Función helper: generar slug desde nombre function generarSlug(nombre) { return nombre .toLowerCase() .normalize('NFD') .replace(/[\u0300-\u036f]/g, '') // eliminar tildes .replace(/[^a-z0-9\s-]/g, '') // solo alfanumérico .trim() .replace(/\s+/g, '-'); // espacios → guión } // Ejemplos: // "Bushi Food" → "bushi-food" // "Capy Beach" → "capy-beach" // "Casa Las Palmas"→ "casa-las-palmas" // "Capitán Mandy" → "capitan-mandy"
