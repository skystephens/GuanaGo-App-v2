⚡ CAPA 3: AUTOMATIZACIÓN (Make.com)

Workflow: Nueva Cotización

Trigger → Notificar Proveedor vía WhatsApp + Email

Flujo Make.com: 1. TRIGGER: Nuevo registro en CotizacionesGG 2. FILTRO: Estado = "Pendiente" 3. BUSCAR: Datos del proveedor (tabla Proveedores) 4. ENVIAR: WhatsApp vía Twilio/WATI Mensaje: "Nueva cotización #COT-1234 por $1,890" 5. ENVIAR: Email con template HTML 6. ACTUALIZAR: Campo "Notificado" = true Costo: ~10 operaciones/cotización Free tier: 1,000 ops/mes = 100 cotizaciones gratis

Workflow: Aprobación

Proveedor aprueba → Notificar Agencia + Crear Booking

Flujo Make.com: 1. TRIGGER: Registro actualizado en CotizacionesGG 2. FILTRO: Estado cambió a "Aprobada" 3. OBTENER: Datos agencia (email, servicio, precio) 4. GENERAR: PDF de cotización (módulo PDF) 5. ENVIAR: Email a agencia con PDF adjunto 6. CREAR: Registro en tabla Bookings 7. NOTIFICAR: Slack interno "Nueva reserva confirmada" Costo: ~15 operaciones/aprobación

Workflow: Sync Precios

Actualización diaria de precios desde Google Sheets

Flujo Make.com: 1. TRIGGER: Scheduled (cada día 6 AM) 2. LEER: Google Sheets con precios actualizados (Proveedores editan aquí manualmente) 3. ITERAR: Por cada fila del Sheet 4. BUSCAR: Servicio en Airtable por ID_Servicio 5. ACTUALIZAR: Campo Precio_Base_Noche 6. LOGGING: Guardar cambios en tabla AuditLog 7. NOTIFICAR: Slack "Precios actualizados: 47 servicios" Costo: ~300 operaciones/día (1 por servicio + overhead) Con plan Core ($10.59/mes): 10,000 ops = suficiente

Workflow: Claude Code Deploy

Claude completa tarea → Trigger CI/CD automático

Flujo Make.com: 1. TRIGGER: Registro actualizado en Tasks_Kanban 2. FILTRO: Estado cambió a "Done" 3. WEBHOOK: GitHub Actions POST https://api.github.com/repos/.../dispatches Event: "claude-code-completed" 4. GITHUB ACTION: - Run tests - Build production - Deploy to Vercel/Render 5. ACTUALIZAR: Airtable con deployment URL 6. NOTIFICAR: Discord/Slack con changelog Costo: ~8 operaciones/tarea completada
